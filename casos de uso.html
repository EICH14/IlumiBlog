¿COMO FUNCIONAN LOS CASOS DE USO EN LOS SISTEMAS DE INFORMACION DE COMERCIO ELECTRONICO?

Los casos de uso en los sistemas de información de comercio electrónico describen las interacciones entre los usuarios (actores) y el sistema para alcanzar un objetivo específico. Estos casos de uso son fundamentales para definir los requisitos funcionales del sistema y asegurar que todas las partes involucradas comprendan cómo se supone que debe funcionar. Aquí hay una descripción de cómo funcionan los casos de uso en un sistema de información de comercio electrónico:

1. Identificación de Actores

Actores: Son los usuarios o sistemas externos que interactúan con el sistema. En un sistema de comercio electrónico, los actores típicos incluyen:
  • Clientes
  • Administradores del sitio
  • Proveedores
  • Pasarelas de pago
  • Sistemas de logística

2. Descripción de Casos de Uso

Cada caso de uso describe una secuencia específica de acciones llevadas a cabo por el actor y el sistema. Aquí hay algunos ejemplos comunes de casos de uso en un sistema de comercio electrónico:
  1. Registro de Usuario
    • Actor: Cliente
    • Descripción: El cliente se registra en el sitio proporcionando información personal.
    • Pasos:
      1. El cliente accede a la página de registro.
      2. El cliente completa el formulario de registro con sus datos.
      3. El cliente envía el formulario.
      4. El sistema valida la información.
      5. El sistema crea una nueva cuenta de usuario y envía una confirmación al cliente.

  2. Iniciar Sesión
    • Actor: Cliente
    • Descripción: El cliente inicia sesión en el sitio utilizando sus credenciales.
    • Pasos:
      1. El cliente accede a la página de inicio de sesión.
      2. El cliente ingresa su nombre de usuario y contraseña.
      3. El cliente envía los datos.
      4. El sistema verifica las credenciales.
      5. El sistema permite el acceso del cliente a su cuenta.

  3. Búsqueda de Productos
    • Actor: Cliente
    • Descripción: El cliente busca productos en el catálogo.
    • Pasos:
      1. El cliente ingresa términos de búsqueda en la barra de búsqueda.
      2. El sistema muestra los resultados de búsqueda relevantes.
      3. El cliente puede filtrar y ordenar los resultados.
      4. El cliente selecciona un producto para ver más detalles.

  4. Añadir al Carrito de Compras
    • Actor: Cliente
    • Descripción: El cliente añade productos a su carrito de compras.
    • Pasos:
      1. El cliente visualiza la página de detalles de un producto.
      2. El cliente selecciona la cantidad deseada.
      3. El cliente hace clic en "Añadir al carrito".
      4. El sistema actualiza el carrito de compras con el producto seleccionado.

  5. Procesar el Pago
    • Actor: Cliente
    • Descripción: El cliente realiza el pago de los productos en el carrito de compras.
    • Pasos:
      1. El cliente revisa el contenido del carrito de compras.
      2. El cliente procede al checkout.
      3. El cliente ingresa la información de envío y pago.
      4. El sistema valida la información y procesa el pago.
      5. El sistema confirma la compra y envía una confirmación al cliente.

3. Documentacion de Casos de Uso

Cada caso de uso se documenta de manera detallada, incluyendo:
  • Nombre del caso de uso
  • Actores involucrados
  • Descripción del objetivo
  • Precondiciones: Lo que debe ser verdadero antes de que el caso de uso pueda iniciar.
  • Postcondiciones: Lo que será verdadero después de que el caso de uso termine.
  • Flujo principal: La secuencia típica de pasos.
  • Flujos alternativos: Variaciones o excepciones en el flujo principal.
  • Requisitos especiales: Consideraciones técnicas o de negocio específicas.

4. Diagramas de Caso de Uso

Para una visualización clara, se suelen utilizar diagramas de casos de uso, que muestran la relación entre los actores y los casos de uso. Estos diagramas ayudan a comprender la estructura general del sistema y cómo interactúan las diferentes partes.


5. Validacion y Refinamiento

Los casos de uso se revisan y refinan en colaboración con las partes interesadas para asegurar que todos los requisitos y escenarios posibles están cubiertos. Esta etapa es crucial para identificar omisiones o errores antes del desarrollo del sistema.


6. Implementacion y Pruebas

Durante la implementación, los desarrolladores utilizan los casos de uso como guía para construir el sistema. Luego, durante las pruebas, se verifican los casos de uso para asegurar que el sistema funcione según lo especificado y cumpla con los objetivos definidos.
